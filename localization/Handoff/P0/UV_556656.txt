Tutorial: Create your own measures in Power BI Designer

	<div><font face="Times New Roman" size="3">

</font><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">Some of the most powerful data analysis solutions in Power
BI Designer can be created by using measures. Measures help us by performing
calculations on our data as we interact with our reports. This tutorial will
guide you through understanding and creating some of your own basic measures in
Power BI Designer.</font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3"></font><br><font face="Calibri" size="3"> </font><br></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3">This article is intended for Power BI users already familiar
with using the designer to create more advanced models. You should already be
familiar with using Query to import data, working with multiple related tables,
and adding fields to the Report Canvas. If you’re new to the designer, be sure
to check out <a href="http://support.powerbi.com/knowledgebase/articles/471664-getting-started-with-power-bi-designer">Getting Started with Power BI Designer</a>.</font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">To complete the steps in this tutorial, you’ll need to download
the <a href="http://download.microsoft.com/download/4/6/A/46AB5E74-50F6-4761-8EDB-5AE077FD603C/Contoso Sales for Power BI Designer.zip">Contoso Sales for Power BI Designer</a> file. It already includes
online sales data from the fictitious company, Contoso, Inc. Because data in
the file was imported from a database, you won’t be able to connect to the
datasource or view it in Query view. When you have the designer file on your
own computer, go ahead and open it in Power BI Designer. </font></p><div><font face="Times New Roman" size="3">

</font></div><h2>First-things-first: What are these measures all about?</h2><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">We create measures when in Power BI Designer’s Report view.</font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">Measures are most often created for us automatically, like
when we select the checkbox next to the <b>SalesAmount</b> field from the <b>Sales</b>
table in the field list, or drag <b>SalesAmount</b> onto the Report canvas.</font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><span><img src="https://powerbi.uservoice.com/assets/79173390/MeasuresTut_SalesAmountInFieldList.png"></span><br></font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">A new chart visualization appears, like this:</font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><span><img src="https://powerbi.uservoice.com/assets/79173402/MeasTut_SalesAmountChart.png"></span></font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">What we get is a Column chart showing a sum total amount of
sales values from the SalesAmount field. <span style="mso-spacerun: yes;">&nbsp;</span>Our SalesAmount field is really just a column
named SalesAmount in the Sales table we already<span style="mso-spacerun: yes;">&nbsp;
</span>imported by using Query. </font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">The SalesAmount column contains over two million of rows of
sales values. You might be wondering why you don’t see a table with rows of all
those values. Well, the designer knows that all of the values in SalesAmount
are of a numeric datatype, and you’ll probably want to aggregate them in some
way, whether it be adding them up, averaging, counting, etc.. </font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">Whenever you see a field in the field list with a sigma icon
<span><img src="https://powerbi.uservoice.com/assets/79173456/MeasTut_Sigma.png"></span><span style="mso-no-proof: yes;"></span>,&nbsp;it means the field is numeric,
and its values can be aggregated. In this case, when we select SalesAmount, the
designer creates its own measure and the sum of all sales amounts is calculated and
displayed in our chart. </font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">Sum is the default aggregation when we select a field with a
numeric datatype, but we can change to a different type of aggregation quite
easily.</font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">In the <b>Value</b> area, if we click the down arrow next to <b>SalesAmount</b>,
we can then select <b>Average</b>.</font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><span><img src="https://powerbi.uservoice.com/assets/79174077/MeasTut_SalesAmountAverage.png"></span></font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">Our visualization changes to an average of all sales values
in the SalesAmount field.</font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><span><img src="https://powerbi.uservoice.com/assets/79174089/MeasTut_SalesAmountAverageChart.png"></span></font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">We can change the type of aggregation depending on the
result we want, but not all types of aggregation apply to just any numeric
datatype. For example, for our SalesAmount field, Sum and Average make sense.
Minimum and Maximum have there place as well. But, Count won’t really make much
sense for our SalesAmount field because while its values are numeric, they’re
really currency. </font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">Understanding aggregations&nbsp;is fundamental to understanding
measures, because every measure will perform some type of aggregation. We’ll see
more examples of using a Sum aggregation a little later, when you create some
of your own measures.</font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">Values calculated from measures are always changing in
response to our interactions with our report. For example, if we drag the <b>RegionCountryName</b>
field from the <b>Geography</b> table to our chart, sales amounts for each country are
averaged and displayed.</font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><span><img src="https://powerbi.uservoice.com/assets/79173432/MeasTut_SalesAmountAvChartByRCN.png"></span></font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">When the result of a measure changes because of an
interaction with our report, we are affecting our measure’s <i>context</i>. In fact,
every time you interact with your report, you are changing the context in which
a measure calculates and displays its results.</font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">In most cases, the designer does its thing and calculates
and returns values according to the fields we’ve added and the types of
aggregation we choose. But in other cases, you might have to create your own
measures to perform more complex, unique calculations. </font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">With Power BI Designer, you create your own measures with the
Data Analysis Expressions (DAX) formula language. DAX formulas are very similar
to Excel formulas. In fact, DAX uses many of the same functions, operators, and
syntax as Excel formulas. However, DAX’s functions are designed to work with
relational<span style="mso-spacerun: yes;">&nbsp;</span>data and perform more dynamic
calculations as we interact with our reports. </font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">There are over 200 DAX functions that do everything
from<span style="mso-spacerun: yes;"> </span>simple aggregations like Sum and
Average to more complex statistical and filtering functions. We’re not going to
go into too much detail on the DAX language here, but there are many resources
to help you learn more. After you've gone through this tutorial, be sure to see QuickStart: DAX in Power BI Designer. </font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">When we create our own measures, they’re added to the field
list for the table we want. This is known as an <i>explicit</i> measure, and it will
remain in our table as a field. Some of the great advantages to explicit
measures are that we can name them what we want, making them more identifiable.
We can also use them as an argument in other DAX expressions, and we can create
measures that perform complex calculations very quickly. </font></p><div><font face="Times New Roman" size="3">

</font></div><h2>Let’s create our own measure</h2><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">Let’s say we want to analyze our net sales. If we look at
our Sales table in the field list, we see that there's no field named
NetSales. But, we have the building blocks to create our own measure to
calculate net sales. </font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">We need a measure to subtract discounts and returns from
sales amounts. Because we want our measure to calculate a result for whatever
context we have in our visualization, in-effect, we need to subtract the sum of
DiscountAmount and ReturnAmount from sum of SalesAmount. This might seem a
little confusing at the moment; don’t worry, it will be more clear in a little
bit.</font></p><div><font face="Times New Roman" size="3">

</font></div><h3>Net Sales</h3><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><b>1.</b>&nbsp; Right click, or click the down arrow on the <b>Sales</b> table in
the field list, and then click <b>New Measure</b>. This will make sure our new measure
is saved in the Sales table, where it will be easier to find.</font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><span><img src="https://powerbi.uservoice.com/assets/79173498/MeasTut_NetSales_NewMeasure.png"></span></font></p><div><font face="Times New Roman" size="3">

</font></div><blockquote><p><font face="Calibri" size="3"><b>Tip: </b>You can also create a new measure by clicking on the
New Measure button in the ribbon on Power BI Designer’s Home tab. </font></p><p><font face="Calibri" size="3"><span><img src="https://powerbi.uservoice.com/assets/79174158/MeasTut_NetSales_NewMeasureRibbon.png"></span></font></p></blockquote><div><font face="Times New Roman" size="3">

</font></div><blockquote><font face="Calibri" size="3">When you create a measure from the ribbon, the measure could
be created in any of the tables. While a measure doesn’t have to belong in a
particular table, it will be easier to find if you create them in a table most
logical to you. If you want it to be in a particular table, click the table
first, to make it active. Then click New Measure. In our case, we’re going to
create our first measure in the Sales table. </font></blockquote><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><br></font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3">The formula bar appears along the top of the Report Canvas.
This is where we can rename our measure and enter a DAX formula.</font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><span><span></span></span></font><font face="Calibri" size="3"><span><span><img src="https://powerbi.uservoice.com/assets/80657790/MeasTut_NetSales_NewMeasure_FormulaBar.png"></span><br></span></font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">Let’s give our new measure a name. By default a new measure
is simply named Measure. If we don’t rename it, when we create another, it will
be named Measure 2, Measure 3, and so on. We want our measures to be more
identifiable, so let’s name our new measure Net Sales. </font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><b>2.&nbsp; </b>Highlight <b>Measure</b> in the formula bar, and then type <b>Net
Sales</b>.</font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">Now we can begin entering our formula. </font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><b>3.</b>&nbsp; After the equals sign type an <b>S</b>. You’ll see a dropdown suggestion
list appear with all of the DAX functions beginning with the letter S. The more
we type, the more the suggestion list is scaled closer to the function we need.
Select<b> SUM </b>by scrolling down, and then press Enter.</font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><span><img src="https://powerbi.uservoice.com/assets/80657637/MeasTut_NetSales_NewMeasure_Formula_S.png"></span></font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">After we press Enter, an opening parenthesis appears along
with another suggestion list of all of the available columns we can pass to the
SUM function.</font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><span><img src="https://powerbi.uservoice.com/assets/80657247/MeasTut_NetSales_NewMeasure_Formula_SUM.png"></span></font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">An expression always appears between an opening and closing parenthesis.
In this case, our expression is going to contain a single argument to pass to
the SUM function; a column to sum up. We can narrow down our list of columns by
typing the first letters of what we want. In this case, we want the SalesAmount
column, so when we begin typing salesam, our list gets smaller, and we are
shown two items we can select. They’re actually the same column. One just shows
[SalesAmount], because we’re creating our measure in the same table the
SalesAmount column is in. The other, we see the table name preceding the column
name.</font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><span><img src="https://powerbi.uservoice.com/assets/79173615/MeasTut_NetSales_NewMeasure_Formula_salesam.png"></span></font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">In general, it’s good practice to enter the fully qualified
name of a column. It will make your formulas easier to read.</font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><b>4.&nbsp;</b>&nbsp;Select <b>Sales[SalesAmount]</b>, and then type a closing parenthesis.</font></p><div><font face="Times New Roman" size="3">

</font></div><blockquote><font face="Calibri" size="3"><b>Tip:</b> Syntax errors are most often caused by a missing or
misplaced closing parenthesis.</font></blockquote><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">Now we want to subtract our other two columns.</font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><b>5.</b>&nbsp; After the closing parenthesis for our first expression, type
a space, and then a minus operator (<b>-</b>), followed by another space. Then enter
another SUM function with the <b>Sales[DiscountAmount]</b> column as its
argument.</font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><span><img src="https://powerbi.uservoice.com/assets/79173705/MeasTut_NetSales_NewMeasure_Formula_DiscAmount.png"></span></font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">We’re starting to run out of space for our formula. No
problem. </font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><b>6.</b>&nbsp; Click the down chevron on the right side of the formula bar.</font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><span><img src="https://powerbi.uservoice.com/assets/79173672/MeasTut_NetSales_NewMeasure_Formula_Chevron.png"></span></font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">Now we have more space. We can enter new parts to
our formula on a new line by pressing Alt-Enter. We can also move things over
by using Tab.</font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><span><span><img src="https://powerbi.uservoice.com/assets/79173714/MeasTut_NetSales_NewMeasure_Formula_Expanded.png"></span></span></font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">Now we can add the final part of our formula.</font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><b>7.</b>&nbsp;&nbsp;Add another minus operator followed by another SUM function
and the <b>Sales[ReturnAmount]</b> column as its argument.</font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><span><img src="https://powerbi.uservoice.com/assets/79173738/MeasTut_NetSales_NewMeasure_Complete.png"></span></font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">Our formula now looks ready.</font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><b>8.</b>&nbsp; Press Enter or click the checkmark in the formula bar to
complete. The formula is validated and added to the field list in the
Sales table.</font></p><div><font face="Times New Roman" size="3">

</font></div><h3>Let’s add our new measure to a report</h3><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">Now we can add our Net Sales measure to the report canvas,
and net sales will be calculated for whatever other fields we add to the
report. Let’s look at net sales by country.</font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><b>1.</b>&nbsp; Drag the <b>Net Sales </b>measure from the <b>Sales</b> table onto the
Report canvas.</font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><b>2.&nbsp;</b> Now drag the <b>RegionCountryName</b> field from the <b>Geography</b> table
into the chart.</font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><span><img src="https://powerbi.uservoice.com/assets/79173753/MeasTut_NetSales_ByRCN.png"></span></font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">Let’s add some more data.</font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><b>3.</b>&nbsp; Drag the <b>SalesAmount</b> field into the chart, to see the
difference between net sales and sales amount. </font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">We now really have two measures in our chart. SalesAmount,
which was summed up automatically by the designer, and the Net Sales measure we
created. In each case, the results were calculated in context of another field
we have in the chart, the countries in RegionCountryName. </font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><span><img src="https://powerbi.uservoice.com/assets/79173774/MeasTut_NetSales_ByRCNandSalesAmount.png"></span></font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">Let’s add a Slicer, so we can further break down our net
sales and sales amounts by calendar year.</font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><b>4.</b>&nbsp; Drag the <b>Year</b> field from the <b>Calendar</b> table next to your
chart.</font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><span><img src="https://powerbi.uservoice.com/assets/79173798/MeasTut_NetSales_YearChart.png"></span></font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">Because Year is a numeric field, the designer summed up its
values and gave us a chart. But, that doesn’t do us much good as a Slicer. </font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><b>5.</b>&nbsp; Click the <b>Change Visualization</b> icon, then click the <b>Table</b>
visualization.</font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><span><img src="https://powerbi.uservoice.com/assets/79173822/MeasTut_NetSales_YearToTable.png"></span></font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><b>6.&nbsp;</b> In <b>Fields</b>, click the down arrow next to <b>Year</b>, and then
click <b>Do Not Summarize</b>. </font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><span><img src="https://powerbi.uservoice.com/assets/79174221/MeasTut_NetSales_Year_DoNotSummarize.png"></span></font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">Now we can change the Year field in the table visualization
into a Slicer.</font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><b>7.</b>&nbsp; Click the <b>Change Visualization</b> icon, and then click the <b>Slicer</b>
visualization.</font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><span><img src="https://powerbi.uservoice.com/assets/79173840/MeasTut_NetSales_Year_ChangeToSlicer.png"></span></font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">Now we have Year as a Slicer. We can select any
individual or group of years and our report’s visualizations will all be sliced
accordingly. </font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><span><img src="https://powerbi.uservoice.com/assets/79173846/MeasTut_NetSales_Year_AsSlicer.png"></span></font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><b>8.&nbsp;</b> Go ahead and click on <b>2013</b>. You’ll see the chart change. Our
Net Sales and SalesAmount measures are re-calculated, showing new results just
for 2013. Here again, we’ve changed the context in which our measures calculate
and display results.</font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><span><img src="https://powerbi.uservoice.com/assets/79173870/MeasTut_NetSales_ChartSlicedByYear.png"></span></font></p><div><font face="Times New Roman" size="3">

</font></div><h2>Let’s create another measure</h2><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">Now that you know how to create your own measures, let’s
create a another. </font></p><div><font face="Times New Roman" size="3">

</font></div><h3>Net Sales per Unit</h3><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">What if we want to find out which products with the most
sales per unit sold is? </font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">Well, we can create another measure. In this case, we want
to divide net sales by the quantity of units sold. In-effect, we want to divide
the result of our Net Sales measure by the sum of Sales[SalesQuantity].</font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><b>1.</b>&nbsp; Create a new measure named <b>Net Sales per Unit </b>in either the Sales
or Products table. </font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">In this measure, we’re going to use the Net Sales measure we
created earlier. With DAX, we can reference other measures in our formula. </font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><b>2.</b>&nbsp; Begin typing <b>Net Sales</b>. The suggestion list will show what
we can add. Select <b>[Net Sales]</b>.</font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><span><img src="https://powerbi.uservoice.com/assets/79173915/MeasTut_NSPU_FormulaStep2A.png"></span></font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">You can also reference another measure by just typing an
opening bracket (<b>[</b>). The suggestion list will show us only the measures we can
add to our formula.</font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><span><img src="https://powerbi.uservoice.com/assets/79174296/MeasTut_NSPU_FormulaStep2B.png"></span></font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><b>3.</b>&nbsp; Right after <b>[Net Sales]</b>, enter a space, then a divide
operator (<b>/</b>), then enter a SUM function, then type <b>Quantity</b>. The suggestion
list shows all of the columns with Quantity in the name. Select <b>Sales[SalesQuantity]</b>.
The formula should now look like this:</font></p><div><font face="Times New Roman" size="3">

</font></div><blockquote><font face="Calibri" size="3"><b>Net Sales per Unit = [Net Sales] / SUM(Sales[SalesQuantity])</b></font></blockquote><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><br></font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3">Pretty cool, huh? Entering DAX formulas is really quite easy
when we use the DAX Editor’s search and suggestion functionality. Now, let’s
see what we get with our new Net Sales per Unit measure.</font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><b>4.&nbsp;</b> Drag the <b>Net Sales per Unit</b> measure onto a blank area in
the report canvas.</font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><span><img src="https://powerbi.uservoice.com/assets/79173945/MeasTut_NSPU_Chart.png"></span></font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">Not very interesting is it? Don’t worry.</font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><b>5.</b>&nbsp; Change the chart visualization type to <b>Tree Map</b>.</font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><span><img src="https://powerbi.uservoice.com/assets/79173960/MeasTut_NSPU_ChangeToTreeMap.png"></span></font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><b>6.&nbsp; </b>Now drag the <b>ProductCategory</b> field from the <b>Products</b> table
down into the <b>Group</b> area.</font></p><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><span><img src="https://powerbi.uservoice.com/assets/79173972/MeasTut_NSPU_ByProductCat.png"></span></font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">That’s some good info, but what if we want to look at net
sales by product?</font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3"><b>7.&nbsp;</b> Remove the <b>ProductCategory</b> field, and then drag the <b>ProductName</b>
field from the <b>Products</b> table down into the <b>Group</b> area instead. <span><img src="https://powerbi.uservoice.com/assets/79173987/MeasTut_NSPU_ByProductName.png"></span></font></p><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">Now that’s just cool!</font></p><div><font face="Times New Roman" size="3">

</font></div><h2>What we’ve learned</h2><div><font face="Times New Roman" size="3">

</font></div><p style="margin: 6pt 0in;"><font face="Calibri" size="3">Measures give us a lot of power in getting the insights we want from our data. You’ve learned how to create measures by using the formula
bar. You can name measures whatever makes most sense to you, and the suggestion
lists make it easy to find and select the right element to add to your
formulas. You’ve also been introduced to context, where the result of
calculations in measures change according to other fields, or by other
expressions in your measure formula.</font></p><div><font face="Times New Roman" size="3">
